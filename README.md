**Project Nexus** is an automated, enterprise-grade Cyber Threat Intelligence (CTI) and Vulnerability Management (VM) platform. It ingests global threat feeds, cross-references them against internal asset inventories, and utilizes Large Language Models (LLMs) to generate boardroom-ready Persian threat reports via XeLaTeX.

## 🏗️ Architecture

The system utilizes a **Hybrid Microservices Architecture**:
1. **Python / FastAPI (The Engine):** Handles heavy data processing, parsing complex NVD JSON APIs, asset matching algorithms, and local XeLaTeX PDF compilation.
2. **PostgreSQL (The Memory):** Stores historical CVE data, prevents duplicate alerts, and powers SOC dashboards.
3. **n8n (The Brain):** Acts as the central orchestrator. It triggers the Python API, queries the database, queries the AI Agent for Persian translation, structures the document using Javascript, and distributes the final PDF via SMTP.

## ✨ Core Features (Phase 1 & 2)

*   **NVD Telemetry Ingestion:** Automatically fetches high-severity (CVSS > 8.0) CVEs published in the last 24 hours.
*   **Intelligent Asset Correlation:** Matches extracted CPE (Common Platform Enumeration) strings against internal CMDB / asset lists.
*   **AI-Powered Localization:** Uses an LLM (e.g., GPT-OSS-20B) to translate highly technical vulnerability descriptions and mitigation strategies into professional Persian.
*   **Zero-Error LaTeX Compilation:** Dynamically generates flawless, bi-directional Persian/English PDF reports using `XePersian` and `Vazirmatn` fonts.
*   **Automated Alerting:** Distributes generated reports directly to System Administrators via Email/Telegram.

## ⚙️ Installation & Setup

### 1. Prerequisites
*   Ubuntu / Debian Linux (Recommended)
*   Python 3.11+
*   PostgreSQL 14+
*   n8n (Docker)
*   XeTeX (TeX Live) with Arabic/Persian language packs

### 2. System Dependencies (Linux)
```bash
sudo apt update
sudo apt install python3-pip python3-venv postgresql postgresql-contrib
sudo apt install texlive-xetex texlive-lang-arabic
```

### 3. Database Setup
Create a PostgreSQL database and user for the project. Make sure to edit `pg_hba.conf` and `postgresql.conf` if n8n is connecting from a Docker subnet (e.g., `172.17.0.0/16`).

### 4. Python Environment Setup
```bash
git clone <repository_url>
cd Project_Nexus
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 5. Configuration
Create a `.env` file in the `config/` directory:
```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=postgres
DB_USER=postgres
DB_PASSWORD=YourSecurePassword
```
*Note: Ensure `.env` is added to your `.gitignore` file.*

## 🚀 Usage

### 1. Start the API Microservice
The Python backend must be running for n8n to communicate with it. It is recommended to run this via `systemd` in production.
```bash
uvicorn --app-dir src api_server:app --host 0.0.0.0 --port 8000
```

### 2. Import n8n Workflow
1. Open your n8n instance.
2. Go to **Workflows** -> **Import from File**.
3. Select `n8n_workflows/nexus_daily_pipeline.json`.
4. Configure your **PostgreSQL** and **SMTP** credentials inside n8n.
5. Set the workflow to **Active**.
